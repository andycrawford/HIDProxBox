HIDProxBox is a Raspberry Pi KVM proxy that routes keyboard and mouse input to up to four computers simultaneously — over USB or Bluetooth. Each target sees a genuine HID device. No drivers. No agents. No software to install on anything you want to control.

Feature Highlights

Everything built into the daemon, no third-party services required.

4-port instant switching

Dedicated selector buttons with LED confirmation. Every computer stays USB-connected at all times — zero warm-up delay on switch.

Wired or wireless input

Accept keystrokes from a USB keyboard or any Bluetooth keyboard. Toggle the active source with a single button press, or long-press to pair a new wireless keyboard on the fly.

USB and Bluetooth output

Relay input via dedicated CH552T HID devices over USB, or turn the Pi into a Bluetooth keyboard for cable-free control. Per-computer output mode persists across reboots.

Browser-based control panel

A built-in web UI at http://<pi-ip>:8080 lets you switch computers and toggle input/output modes from any phone, tablet, or browser — no app required.

Optional touchscreen display

A local tkinter GUI runs on any Pi-compatible touchscreen: tally lights and one-tap switching, with a built-in ZSA Keymapp launcher for ZSA keyboard users.

Programmable HID macros

Map keystroke sequences (lock screen, screenshot, custom combos) to button short- or long-press actions via a plain Python config file.

Nothing to install on targets

Each CH552T enumerates as a standard USB HID keyboard and mouse. The Pi's Bluetooth output appears as a standard wireless keyboard. Your computers never know it is a proxy.

Fully open-source and hackable

Python daemon with a clean producer-consumer architecture. Add new input sources or output sinks in a few lines of code. MIT licensed — build it, fork it, sell it, improve it.

Who It's For

Real workflows, no sales pitch.

01

The Multi-Machine Developer

Beefy workstation on the left, work laptop docked to the right, headless server in the rack. A single ergonomic keyboard across all three — no USB unplugging, no KVM lag, no extra software on the laptop you don't own.

Press button 1 for the workstation, button 2 for the laptop (Bluetooth, so it's still on the docking station's USB ports), hold a button to fire SSH macros straight into the server. Every switch is instant. Every machine thinks it has a dedicated keyboard.

02

The Clean-Desk Home Office

Personal Mac beside a company-issued Windows PC. Software KVMs break after updates, need admin rights, and IT policy won't allow them. HIDProxBox is purely hardware from the targets' perspective.

Both machines are always plugged in, always ready. One physical button (or a tap on the web panel from your phone) hands input from one to the other. Toggle Bluetooth output and your Mac never needs a USB cable at all.

03

The Maker Workbench

Building and flashing firmware across a Pi, an STM32 discovery board, and an x86 test machine — all headless, all needing occasional keyboard input for setup and logs.

HIDProxBox lives on a shelf, wired into all three. Pair your Bluetooth mechanical keyboard as the input source — no cable to knock loose. Add macro buttons for your most-used debug commands. Connect the optional touchscreen for a clean, self-contained control panel.

Get Started

Clone, install, run. Flash the CH552T Arduino sketch. Wire buttons and LEDs to GPIO. Done.

1

Clone and install

git clone https://github.com/andycrawford/HIDProxBox.git /opt/hidproxbox
pip3 install evdev pyserial pybluez --break-system-packages
sudo apt-get install bluetooth bluez python3-dbus python3-gi
2

Flash the firmware

Flash firmware/hidproxbox.ino to each CH552T unit using the Arduino IDE with CH55x board support. One CH552T per computer.

3

Wire and configure

Connect CH552T units via USB-serial adapters. Wire tactile buttons and LEDs to GPIO. Update serial ports, GPIO pin numbers, and macros in config.py.

4

Run the daemon

sudo python3 /opt/hidproxbox/daemon.py --foreground

# Or install as a systemd service
sudo systemctl start hidproxbox
sudo systemctl enable hidproxbox
